# Подключение ИИ-агента к Linear

ИИ-агент может работать с задачами Linear через GraphQL API: читать задачи, добавлять комментарии, менять статусы и создавать новые задачи.

## Быстрое подключение через API-токен

Этот вариант подходит для личного агента, тестирования и внутренних автоматизаций.

1. В Linear откройте **Settings → Security & access → Personal API keys**.
2. Создайте ключ и сразу сохраните его: повторно Linear его не покажет.
3. Передайте ключ агенту через переменную окружения, а не напрямую в коде:

   ```bash
   export LINEAR_API_KEY="lin_api_..."
   ```

4. Настройте агента на отправку GraphQL-запросов по адресу:

   ```text
   https://api.linear.app/graphql
   ```

5. Добавляйте ключ в заголовок каждого запроса:

   ```text
   Authorization: lin_api_...
   Content-Type: application/json
   ```

6. Проверьте подключение запросом текущего пользователя:

   ```bash
   curl https://api.linear.app/graphql \
     -H "Authorization: $LINEAR_API_KEY" \
     -H "Content-Type: application/json" \
     --data '{"query":"query { viewer { id name email } }"}'
   ```

Если подключение настроено правильно, Linear вернёт данные пользователя в поле `data.viewer`.

## Подключение нескольких пользователей через OAuth

Для общего агента или продукта, которым будут пользоваться разные участники и рабочие пространства:

1. Создайте OAuth-приложение в настройках Linear.
2. Укажите разрешённый callback URL приложения.
3. Запросите только необходимые агенту права доступа.
4. Реализуйте OAuth 2.0-авторизацию и безопасное хранение полученных токенов.
5. Используйте access token в заголовке `Authorization: Bearer <token>`.

## Получение событий из Linear

Если агент должен автоматически реагировать на создание или изменение задач, настройте webhook:

1. Создайте публичный HTTPS endpoint в приложении агента.
2. Добавьте webhook в настройках Linear и выберите нужные типы событий.
3. Проверяйте подпись каждого входящего webhook-запроса.
4. Быстро отвечайте успешным HTTP-статусом, а обработку события выполняйте асинхронно.
5. Учитывайте, что одно событие может быть доставлено повторно.

## Рекомендации по безопасности

- Не добавляйте API-ключи и OAuth-токены в Git, логи или текст задач.
- Храните секреты в переменных окружения или в менеджере секретов.
- Выдавайте агенту минимально необходимые права.
- Запрашивайте подтверждение пользователя перед удалением задач и другими необратимыми действиями.
- Ограничьте доступ агента нужными командами и проектами на уровне его логики.
- Регулярно меняйте токены и сразу отзывайте скомпрометированные ключи.
